---
title: Status
description: Inspect Studio runtime status, storage adapters, counts, and capabilities.
---

The Status page gives a compact operational snapshot of the local Studio runtime.

Open `http://localhost:4021/ui/status` after starting Studio.

## HTTP Route

```bash
curl http://localhost:4021/status
```

```json
{
  "runner": {
    "id": "anvia-studio"
  },
  "storage": {
    "sessions": "memory",
    "traces": "memory",
    "pipelineLogs": "available",
    "pipelineRuns": "available"
  },
  "counts": {
    "agents": 1,
    "pipelines": 0,
    "sessions": 3,
    "traces": 5
  },
  "capabilities": {
    "agents": { "enabled": true },
    "status": { "enabled": true },
    "sessions": { "enabled": true },
    "memory": { "enabled": true },
    "traces": { "enabled": true }
  },
  "generatedAt": "2026-06-01T00:00:00.000Z"
}
```

Counts are capped to the current Studio list limits, so use them as a quick local signal rather than a production metric source.
